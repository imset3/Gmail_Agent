Gmail Multi-Agent Email Assistant

실제 Gmail API 또는 더미 메일 데이터를 사용해 메일을 병렬 분류하고, 자동답장·스팸 등록·일정 충돌·사용자 판단 필요 항목을 처리하는 과제 제출 문서입니다.

개요 실행 방법 구조 처리 흐름 Windows Gmail API 산출물 오류 해결 GitHub 제출

1. 프로젝트 개요

이 프로젝트는 메일함을 자동으로 분석하는 멀티 Agent 시스템입니다. 입력은 더미 JSON 메일 또는 실제 Gmail API이며, 출력은 콘솔 리포트와 JSON/Markdown 리포트입니다.

Dummy Mail Gmail API Multi-Agent LLM/Ollama/OpenAI 교체 가능 Windows 지원
스팸광고, 이벤트, 의심 발신자를 탐지하고 data/spam_db.json에 등록합니다.
무응답공지, 시스템 알림, 단순 안내 메일은 답장 불필요로 분류합니다.
의사결정 필요일정 충돌, 예산안, 승인 요청처럼 사용자의 판단이 필요한 항목을 따로 모읍니다.
자동답장답장 가능한 메일은 더미 발신함 또는 Gmail Draft 초안으로 저장합니다.

2. 실행 방법

macOS/Linux

cd Gmail_Agent
python3 start.py

Windows

압축을 풀고 RUN_WINDOWS.bat을 더블클릭하면 메뉴가 실행됩니다.

RUN_WINDOWS.bat

명령어로 바로 실행

파일/폴더역할

Agent 역할

  • MailFetchAgent: 더미 JSON 또는 Gmail API에서 메일을 가져옵니다.
  • ClassificationAgent: LLM 또는 규칙 기반 fallback으로 메일을 분류합니다.
  • SpamAgent: 스팸 발신자를 DB에 저장합니다.
  • ScheduleAgent: 메일 속 날짜와 기존 일정을 비교해 충돌을 찾습니다.
  • ReplyAgent: 자동답장 결과를 더미 발신함 또는 Gmail Draft로 저장합니다.
  • ReportAgent: 콘솔, JSON, Markdown 리포트를 생성합니다.

4. 처리 흐름

  1. 메일을 가져옵니다. 더미 모드는 dummy_mails/emails.json, Gmail 모드는 Gmail API를 사용합니다.
  2. asyncio.gather로 메일별 분류 작업을 병렬 실행합니다.
  3. 스팸은 spam_db.json에 등록합니다.
  4. 의사결정 필요 메일은 일정 충돌과 사용자 판단 상태를 기록합니다.
  5. 자동답장 메일은 더미 모드에서 sent_mails.json에 저장하고, Gmail 모드에서는 Draft 초안을 생성합니다.
  6. 최종 결과를 콘솔과 output/results.json, report/email_report.md에 저장합니다.

5. Windows 실행 지원

Windows 사용자는 터미널 명령을 외우지 않아도 됩니다. 압축을 풀고 아래 배치 파일 중 하나를 실행하면 됩니다.

파일용도
RUN_WINDOWS.bat메뉴 실행: 더미/Gmail/인증을 선택합니다.
RUN_DUMMY_WINDOWS.bat더미 메일 모드로 바로 실행합니다.
AUTH_GMAIL_WINDOWS.batGmail OAuth 인증을 실행합니다.
RUN_GMAIL_WINDOWS.batGmail 패키지를 설치한 뒤 실제 Gmail 모드로 실행합니다.
RUN_WINDOWS.ps1PowerShell 환경에서 메뉴를 실행합니다.

주의: Gmail API를 쓰려면 config/credentials.json 파일이 필요합니다. 이 파일은 개인정보라 GitHub에는 올리지 않습니다.

6. Gmail API 적용 방법

  1. Google Cloud Console에서 Gmail API를 활성화합니다.
  2. OAuth Client를 Desktop app으로 생성합니다.
  3. 다운로드한 JSON 파일명을 credentials.json으로 바꿉니다.
  4. config/credentials.json 위치에 넣습니다.
  5. python3 start.py --auth 또는 AUTH_GMAIL_WINDOWS.bat을 실행해 인증합니다.

Gmail 모드에서는 안전을 위해 실제 발송을 하지 않고 Draft 초안만 만듭니다.

8. 발생 가능 오류와 해결

오류원인해결
Python이 설치되어 있지 않습니다 Windows PATH에 Python이 없습니다. Python 3.10 이상 설치 후 Add python.exe to PATH를 체크합니다.
ModuleNotFoundError: google... Gmail API 패키지가 설치되지 않았습니다. pip install -r requirements.txt 또는 RUN_GMAIL_WINDOWS.bat 실행.
config/credentials.json 파일이 필요합니다 Gmail OAuth JSON 파일이 없습니다. 다운로드한 OAuth JSON을 config/credentials.json으로 저장합니다.
403 access_denied OAuth 앱이 Testing 상태인데 테스트 사용자에 Gmail 계정이 없습니다. Google Cloud Console의 OAuth consent screen에서 Test users에 로그인 계정을 추가합니다.
Gmail은 되는데 답장이 발송되지 않음 의도된 안전 정책입니다. 실제 발송 대신 Gmail Draft 초안만 생성합니다. Gmail에서 직접 확인 후 발송하면 됩니다.
LLM이 실행되지 않음 OpenAI/Ollama 환경변수가 없습니다. 기본 규칙 fallback으로 정상 실행됩니다. LLM을 쓰려면 OPENAI_API_KEY 또는 OLLAMA_MODEL을 설정합니다.
GitHub에 인증 파일이 올라갈까 걱정됨 OAuth 파일은 민감 정보입니다. .gitignore에 config/credentials.json, config/token.json을 제외 처리했습니다.
